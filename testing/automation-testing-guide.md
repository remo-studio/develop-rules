# 自动化测试指南

## 测试范围（Scope）
- 覆盖本项目中所有可脚本化、可重复执行的测试场景。
- 优先包括：核心业务流程、关键配置、对外接口、重要数据转换逻辑。
- 不包括：强依赖人工感知的交互体验（如视觉审美、动效流畅度等）。

## 负责人（Owner）
- 测试负责人：测试团队 / QA Owner。
- 自动化脚本维护者：对应模块的开发工程师与测试工程师共同负责。
- CI 管道维护：DevOps / 平台工程团队。

## 验证方法（Verification Method）
- 所有自动化用例必须接入 CI（例如：提交代码后自动执行）。
- 每次执行需生成测试报告（至少包含：通过率、失败用例列表、执行时间）。
- 对关键用例设置“必过”门禁：失败时禁止合并或发布。

## 案例 Sample（Example）
- 用例名称：用户登录成功流程自动化
- 前置条件：
  - 测试环境已部署完成；
  - 测试账号 `test_user` 已存在，密码为预置测试密码。
- 步骤：
  1. 打开登录页。
  2. 输入用户名 `test_user`。
  3. 输入正确密码。
  4. 点击“登录”按钮。
- 期望结果：
  - 返回 HTTP 200；
  - 页面跳转到首页；
  - 显示用户昵称 "Test User"；
  - 自动化脚本断言以上结果均成立。
