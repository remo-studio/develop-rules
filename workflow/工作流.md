# AI辅助开发方案

## 第一部分：各阶段AI生成成果物一览

| 阶段 | 子分类 | AI生成成果物 | 格式 | 说明 |
|------|--------|------------|------|------|
| **市场调研** | 调研 | 市场分析 | Markdown | 行业现状、市场规模、发展趋势 |
| | 调研 | 竞品分析 | Markdown | 竞品功能对比、优劣势分析 |
| | 预测 | 收益预测 | Markdown | 收入模型、成本估算、ROI分析 |
| | 规划 | 产品方向定义 | Markdown | 产品定位、目标用户、核心价值 |
| | 分析 | 风险与决策记录 | Markdown | 风险识别、关键决策、假设记录 |
| **需求定义** | 功能 | PRD文档 | Markdown | 用户故事、业务规则、功能描述 |
| | 模型 | 领域模型 | PlantUML + Markdown | 实体关系、业务建模 |
| | UI | 页面流程图 | PlantUML | 页面跳转与用户路径 |
| **设计** | 架构 | 系统架构图 | PlantUML | 服务划分、模块关系 |
| | 数据 | 数据库设计 | Markdown + SQL | 表结构、ER关系 |
| | 接口 | API设计 | Markdown | 接口协议与规范 |
| | 结构 | 类设计 | Java / PHP | 类结构与职责划分 |
| **开发** | 后端 | 后端代码 | Java | 业务逻辑实现 |
| | 前端 | 前端代码 | Vue + TS | 页面与交互 |
| | 批处理 | 定时任务 | Java / PHP | 数据处理任务 |
| | 数据 | 数据库迁移 | SQL (Flyway) | 版本化变更 |
| **测试** | 单元 | 单元测试 | JUnit / Jest | 方法级验证 |
| | 集成 | 集成测试 | Spring Boot Test | 接口联调测试 |
| | E2E | 端到端测试 | Playwright / Cypress | 全流程业务验证 |
| | 数据 | 测试数据 | SQL | 测试数据准备 |
| **发布** | 环境 | Docker配置 | docker-compose | 环境搭建 |
| | 流程 | 发布流程文档 | Markdown | 发布步骤与检查 |
| | 变更 | 代码提交/PR | Git | 发布变更记录 |

## 第二部分：AI动物角色 × Skills 定义

| 阶段 | 动物角色 | Skills（能力标签） | 说明 |
|------|----------|------------------|------|
| 项目管理 | 🦁 狮子 | project-planning, task-allocation, risk-management, decision-making | 负责整体规划、任务分配与关键决策 |
| 需求定义 | 🦉 猫头鹰 | requirement-analysis, domain-modeling, business-understanding, abstraction | 深入理解需求，抽象业务模型 |
| UI设计 | 🦋 蝴蝶 | ui-design, ux-optimization, component-design, interaction-flow | 负责界面设计与用户体验优化 |
| 系统设计 | 🐘 大象 | system-architecture, database-design, api-design, scalability | 构建系统架构与数据结构 |
| 详细设计 | 🕷 蜘蛛 | detailed-design, class-design, dependency-management, migration-planning | 细化设计与技术实现方案 |
| 开发 | 🐝 蜜蜂 | coding, implementation, refactoring, task-execution | 高效实现功能与代码开发 |
| 测试 | 🐶 猎犬 | testing, bug-detection, validation, qa | 发现问题并保证质量 |
| 发布 | 🦅 老鹰 | deployment, monitoring, release-management, environment-control | 负责发布与运行环境控制 |

### Skills 命名规范说明

- 所有 Skills 使用小写 + kebab-case（如 `project-planning`）
- 每个阶段对应一个 AI Agent，可组合多个 Skills
- Skills 可作为 good7ob 平台中 AI 能力调用的最小单元
- 支持后续扩展（如 logging-analysis / cost-optimization 等）
