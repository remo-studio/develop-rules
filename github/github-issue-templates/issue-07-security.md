```markdown
### 7.1 数据安全要求 - @username

**数据分类**:

| 数据类型 | 分类级别 | 说明 |
|---------|---------|------|
| 用户密码 | 机密数据 | 加密存储,不可逆 |
| 用户手机号/邮箱 | 敏感数据 | 需加密存储 |
| 订单金额/交易记录 | 敏感数据 | 财务相关 |
| 用户行为数据 | 内部数据 | 分析使用 |
| 商品信息 | 内部数据 | 公开部分为公开数据 |

**数据加密要求**:

| 加密类型 | 状态 | 说明 |
|---------|------|------|
| 传输加密 (TLS/SSL) | ✅ 已实施 | 所有外部接口使用 HTTPS |
| 存储加密 | ⚠️ 部分实施 | AWS RDS 启用加密,IDC 数据库未加密 |
| 字段级加密 | ⚠️ 部分实施 | 用户密码、支付信息已加密 |

**安全问题**:
- IDC 数据库未启用存储加密
- 部分敏感字段(如手机号)未加密
```

```markdown
### 7.2 访问控制 - @username

**数据库访问权限控制方式**:

| 数据库 | 控制方式 | 说明 |
|--------|---------|------|
| juren-prod (AWS) | IP 白名单 + IAM 角色 | 严格控制 |
```

```markdown
### 7.3 合规要求 - @username

**需要遵守的合规标准**:

| 合规标准 | 是否适用 | 说明 |
|---------|---------|------|
| GDPR (欧盟数据保护) | ❌ 否 | 暂无欧盟用户 |
| SOC 2 | ❌ 否 | 未要求 |
```
