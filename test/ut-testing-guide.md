# UT 测试指南（单元测试）

## 测试范围（Scope）
- 面向单个函数、类或模块的最小可测试单元。
- 覆盖：
  - 业务核心计算逻辑；
  - 输入输出数据转换；
  - 异常分支与边界条件；
  - 依赖外部服务的模块需通过 Mock/Stubs 隔离。

## 负责人（Owner）
- 主要负责人：各模块的开发工程师。
- 审阅与补充：测试工程师。

## 验证方法（Verification Method）
- 每次代码提交必须通过全部 UT。
- UT 覆盖率（行覆盖/分支覆盖）应达到团队约定基线（例如：行覆盖 ≥ 80%，核心模块 ≥ 90%）。
- 对关键函数的异常场景必须有对应 UT。

## 案例 Sample（Example）
- 测试对象：`calculateDiscount(price, level)`
- 前置条件：
  - 输入 `price` 为正数；
  - `level` 取值范围：`NORMAL`、`VIP`、`SVIP`。
- 用例 1：正常用户
  - 输入：`price = 100`, `level = NORMAL`
  - 期望结果：返回 `100`（无折扣）。
- 用例 2：VIP 用户
  - 输入：`price = 100`, `level = VIP`
  - 期望结果：返回 `90`（9 折）。
- 用例 3：非法参数
  - 输入：`price = -1`, `level = NORMAL`
  - 期望结果：抛出参数异常或返回约定错误码。
